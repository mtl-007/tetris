# 브라우저 테트리스 MVP

> **파일 구성: 단일 HTML 파일이 아니라 `index.html` / `style.css` / `script.js` 3개 파일로 분리하여 작성함.**

## Context
`day01/tetris` 폴더가 비어 있는 상태로 새로 생성됨. 별도 빌드 도구 없이 브라우저에서 바로 열어 실행할 수 있는 간단한 테트리스 게임이 필요하며, 고급 기능(홀드, 다음 블록 미리보기, 레벨업, 고스트 피스 등) 없이 "점수 표시 + 블록 이동/회전 + 줄 삭제 + 게임오버"만 갖춘 MVP면 충분함.

같은 디렉토리의 `personal_landing` 프로젝트가 빌드 도구 없는 순수 HTML/CSS/JS + 다크 모드 카드형 UI(`--color-bg: #15171c`, accent `#6e93ff`, radius 16px 등)를 사용하고 있어, 동일한 톤(다크 배경 + 카드형 레이아웃)으로 디자인을 맞춰 일관성을 유지.

## 구현 범위
순수 HTML/CSS/JS (빌드 도구 없음, `index.html` 더블클릭으로 실행 가능):

- `index.html` — 캔버스(게임판) + 점수 표시 + 시작/재시작 버튼 + 조작법 안내
- `style.css` — `personal_landing`과 동일 계열의 다크 카드 테마
- `script.js` — 게임 로직 전체

## 게임 사양 (MVP)
- 보드: 10열 x 20행, 셀 30px (canvas 300x600)
- 표준 7종 테트로미노(I, O, T, S, Z, J, L), 테트로미노별 고정 색상
- 조작: ← → (이동), ↓ (소프트 드롭), ↑ 또는 Space (회전) — 키보드만 지원
- 자동 낙하: 고정 간격 타이머(예: 800ms)로 한 칸씩 하강
- 충돌 판정 → 줄 완성 시 해당 줄 삭제 + 위 블록 시프트
- 점수: 한 번에 지운 줄 수에 따라 가산 (1줄 100 / 2줄 300 / 3줄 500 / 4줄 800 식의 간단한 표 또는 `lines * 100`)
- 게임오버: 새 피스 스폰 위치가 막혀 있으면 종료, 오버레이로 "Game Over" + 최종 점수 + 재시작 버튼 표시
- 한 화면에 보드 + 점수 패널만 존재 (다음 블록 미리보기 없음)

## 파일별 작업
0. **plan.md**: 구현 시작 전, 이 계획 내용을 `day01/tetris/plan.md` 파일로 저장.
1. **index.html**: 다크 카드(`.card`) 안에 `<canvas id="board">`와 점수 영역(`<div id="score">`), 재시작 버튼, 간단한 조작법 텍스트 배치. `style.css`, `script.js` 연결.
2. **style.css**: `personal_landing/style.css`의 색상 변수(`--color-bg`, `--color-card-bg`, `--color-accent`, `--radius` 등)를 그대로 가져와 톤을 맞추고, 캔버스 테두리/점수 텍스트 스타일만 추가.
3. **script.js**:
   - 보드 상태를 2차원 배열로 관리
   - 7종 피스 정의(모양 행렬 + 색상)
   - 스폰 / 이동 / 회전(충돌 시 회전 취소) / 충돌 검사 / 고정(lock) / 줄 삭제 / 점수 갱신 함수
   - `setInterval` 또는 `requestAnimationFrame` 기반 게임 루프로 자동 낙하 처리
   - `keydown` 이벤트로 좌우/소프트드롭/회전 처리
   - 게임오버 감지 및 재시작 버튼 핸들러

## 검증
- 브라우저에서 `index.html`을 직접 열어(파일 프로토콜) 정상 동작 확인
- 좌/우/회전/소프트드롭 키 입력으로 블록 조작 확인
- 한 줄을 채워 줄 삭제 + 점수 증가 확인
- 보드를 채워 게임오버 오버레이 표시 및 재시작 동작 확인
